# PresenceCV 產品開發藍圖 & 變現規劃 (ROADMAP)

此文件為 PresenceCV 的產品規劃藍圖，旨在協助專案以最快速度驗證商業價值，並逐步整合自動化投遞工具 (104 Job Auto Applier) 實現商業變現。

---

## 🎯 產品變現與收費機制 (Monetization & Tiering)

我們將產品功能簡化為兩個層級：**Free (免費版)** 與 **Pro (專業版)**：

| 功能項目 | Free (免費版) | Pro (專業版) |
| :--- | :---: | :---: |
| **定價** | $0 / 永久 | ☕ $99 NTD / 月 |
| **履歷 Profile 數量** | **2 個** | 無限制 |
| **AI 履歷解析匯入** | 1 次 / 日 | 無限制 |
| **PDF 匯出** | 基礎單頁 A4 (包含浮水印) | 精美無浮水印 / 支援多頁排版 |
| **分享連結** | 快照連結 + **Live 實時同步連結** | 快照連結 + Live 實時同步連結 |
| **104 自動投遞限制** | 每 **2 天** 可手動啟動 1 次<br>(單次上限 **10 個** 工作) | **無限制** 頻率與次數<br>(支援排程自動化與手動啟動) |
| **自動投遞適配度分析** | ❌ **不進行分析**<br>(搜尋到的職缺直接投遞) | 🟢 **分析職缺與履歷適配度並評分**<br>(僅投遞適合的職缺) |
| **客製化求職信 (Cover Letter)** | 🟢 支援 (依職缺自動撰寫) | 🟢 支援 (依職缺自動撰寫) |

## 🗺️ 開發階段規劃 (Development Phases)

### 📌 階段一：104 Job Auto Applier & AI Agent 自動投遞整合
*將履歷工具升級為「主動式求職代理」，依收費規則限制投遞行為。*

- [ ] **1. 自動投遞後端整合 (Puppeteer / Playwright)**
  - 於後端建立基於 Headless 瀏覽器的自動投遞執行器，串接 104 會員登入。
  - 實作自動填寫求職信並送出投遞的自動化腳本。
- [ ] **2. 投遞適配度過濾機制 (Pro 版專屬)**
  - 針對 Pro 用戶，在投遞前呼叫 AI Agent 分析 104 職缺 JD 與履歷適配度，評分高於門檻才執行投遞。
  - 免費版使用者則跳過適配度分析，直接將搜尋到的前 10 個職缺全數投遞。
- [ ] **3. 排程與手動控制面板**
  - 前端新增「AI 投遞控制室」頁面：
    - **免費版**：限制每 2 天可手動點擊「啟動投遞」1 次，單次處理 10 個職缺；禁用排程自動投遞。
    - **Pro 版**：支援手動點擊（無限次）與「每日定時自動投遞」排程。
- [ ] **4. AI 自動求職信生成 (Free/Pro 皆支援)**
  - 針對確定要投遞的職缺，AI 依據 JD 自動為該次投遞撰寫客製化的求職信。

### 📌 階段二：多國語系支援 (Internationalization - i18n)
*新增語系切換功能，提供中英文介面切換，以利全球用戶使用。*

- [ ] **1. 多國語系框架與資源建立**
  - 導入語系管理套件（如 `react-i18next`），並將全站靜態文字抽離至獨立語系設定檔中。
- [ ] **2. 語言切換選單 (EN / 繁中)**
  - 於導覽列或設定選單中，新增簡約質感的語系切換下拉選單。
  - 預設依據用戶瀏覽器語系進行自動判定，並將偏好設定記錄於 localStorage。
